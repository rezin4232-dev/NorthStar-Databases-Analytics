# NorthStar Databases and Analytics Assignment

This repository contains the analytical workflow for the NorthStar Urban Mobility and Logistics case study.

## Project Overview

The project analyses operational performance issues at NorthStar, including delays, failed deliveries, complaints, incidents, hub performance, service costs, and fragmented data systems.

The aim is to identify the causes of operational inefficiency and demonstrate how integrated analytics and MongoDB NoSQL design can support better decision-making.

## Repository Contents

- `01_python_data_cleaning_northstar.ipynb`  
  Python data loading, cleaning, transformation, integration, KPI analysis, and visualisations.

- `02_R_SQL_Analytics_NorthStar.ipynb`  
  SQL queries inside R, R analytics, and visualisations.

- `03_MongoDB_NorthStar.ipynb`  
  MongoDB Atlas connection, NoSQL document design, CRUD operations, aggregation queries, indexing, and explain plan analysis.

- `northstar_integrated_cleaned.csv`  
  Cleaned integrated dataset used for analysis.

## Key Analytical Focus

The analysis focuses on:

- Delay and failed delivery patterns
- Complaint and incident trends
- Hub and service-type performance
- Manual route overrides
- Vehicle maintenance and operational issues
- MongoDB document-based service case design
- Query optimisation using indexes and explain plans
